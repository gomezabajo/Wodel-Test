We thank the developers of the following frameworks and tools, which have supported the development of Wodel-Test:

<div style="text-align: left" markdown="1">
- [![Xtext](../../assets/img/icons/plugins.png){:style='float: left;margin-right: 12px;margin-top: 4px; height:20px; width:20px;'}](https://www.eclipse.org/Xtext/) [Xtext](https://www.eclipse.org/Xtext/): Xtext supports the development of programming languages and domain-specific languages. A language is defined using a concise and expressive grammar language.
</div>
<div style="text-align: left" markdown="1">
- [![Sirius](../../assets/img/icons/plugins.png){:style='float: left;margin-right: 12px;margin-top: 4px; height:20px; width:20px;'}](https://www.eclipse.org/sirius/) [Sirius](https://www.eclipse.org/sirius/): Sirius supports the creation of graphical modelling workbenches using Eclipse Modelling technologies, including EMF and GMF.
</div>
<div style="text-align: left" markdown="1">
- [![Epsilon](../../assets/img/icons/plugins.png){:style='float: left;margin-right: 12px;margin-top: 4px; height:20px; width:20px;'}](https://www.eclipse.org/epsilon/) [Epsilon](https://www.eclipse.org/epsilon/): Epsilon is a family of Java-based scripting languages for automating common model-based software-engineering tasks. These tasks include code generation, model-to-model transformation and model validation. Epsilon works directly with EMF, including Xtext and Sirius, as well as UML, Simulink, XML and other model formats.
</div>
<div style="text-align: left" markdown="1">
- [![OCL Tools](../../assets/img/icons/plugins.png){:style='float: left;margin-right: 12px;margin-top: 4px; height:20px; width:20px;'}](https://projects.eclipse.org/projects/modeling.mdt.ocl) [OCL Tools](https://projects.eclipse.org/projects/modeling.mdt.ocl): Eclipse OCL implements the Object Constraint Language standard defined by the Object Management Group for EMF-based models.
</div>
<div style="text-align: left" markdown="1">
- [![Acceleo](../../assets/img/icons/plugins.png){:style='float: left;margin-right: 12px;margin-top: 4px; height:20px; width:20px;'}](https://www.eclipse.org/acceleo/download.html) [Acceleo](https://www.eclipse.org/acceleo/download.html): Acceleo is a template-based technology with authoring tools for creating custom code generators. It can automatically generate source code from data represented in EMF format.
</div>
<div style="text-align: left" markdown="1">
- [![MoDisco](../../assets/img/icons/plugins.png){:style='float: left;margin-right: 12px;margin-top: 4px; height:20px; width:20px;'}](https://projects.eclipse.org/projects/modeling.modisco) [MoDisco](https://projects.eclipse.org/projects/modeling.modisco): Eclipse MoDisco provides an extensible framework for model-driven software reverse engineering and modernisation. Its use cases include technical migration, software improvement, documentation generation and quality assurance.
</div>
<div style="text-align: left" markdown="1">
- [![ATL](../../assets/img/icons/plugins.png){:style='float: left;margin-right: 12px;margin-top: 4px; height:20px; width:20px;'}](https://www.eclipse.org/atl/) [ATL](https://www.eclipse.org/atl/): ATL, the ATL Transformation Language, is a language and toolkit for model transformation. In model-driven engineering, it is used to generate target models from source models.
</div>
<div style="text-align: left" markdown="1">
- [![AnATLyzer](../../assets/img/icons/plugins.png){:style='float: left;margin-right: 12px;margin-top: 4px; height:20px; width:20px;'}](https://anatlyzer.github.io/) [AnATLyzer](https://anatlyzer.github.io/): AnATLyzer is a static analyser for ATL. It is provided as an Eclipse plug-in that extends the ATL development environment with features intended to improve developer productivity.
</div>
<div style="text-align: left" markdown="1">
- [![Conga](../../assets/img/icons/plugins.png){:style='float: left;margin-right: 12px;margin-top: 4px; height:20px; width:20px;'}](https://saraperezsoler.github.io/CONGA/) [CONGA](https://saraperezsoler.github.io/CONGA/): CONGA is a model-driven engineering approach for developing chatbots.
</div>
<div style="text-align: left" markdown="1">
- [![EMF JSON](../../assets/img/icons/plugins.png){:style='float: left;margin-right: 12px;margin-top: 4px; height:20px; width:20px;'}](https://emfjson.github.io/projects/jackson/latest/) [EMF JSON](https://emfjson.github.io/projects/jackson/latest/): EMF JSON is a family of projects that provides support for JSON and related technologies in the Eclipse Modelling Framework. It offers an alternative to the XML Metadata Interchange format used by default for EMF serialisation.
</div>
<br>

### Authors
<br>
The [Wodel-Test](../../) research project was initiated by the [miso](https://miso.es) research group in October 2017. Since then, [Pablo Gómez-Abajo](https://www.gomezabajo.es/) has led its development under the supervision of [Esther Guerra](https://www.ii.uam.es/~eguerra) and [Juan de Lara](http://arantxa.ii.uam.es/~jlara/). During his doctoral studies, [Mercedes Merayo](http://antares.sip.ucm.es/mercedes/) also contributed to this supervision.
<br>
